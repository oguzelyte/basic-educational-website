LESSON 1

HTML and its structure

The structure of HTML is a mandatory requirement since you don't have that much control as you would have compared with printed material. It is necessary to keep a good style, because it helps various browsers render it consistently, users can fully exploit your website, editing and maintaining is easier for you and your coding team (assuming you have one). Page authors get full access to the style sheet and various scripting features.

Web page specifications

All of the specs are provided by the W3C (the page also has very nicely commented code snippets teaching you the ropes and more). W3C publishes DTDs for each version of HTML they release, it shortly tells which tags can be used, where they are allowed to appear and what they can hold inside themselves. The DTD also tells how web browsers are supposed to render its code.

Margaret talks how website design is made and how it affects the many

An HTML document is composed of three parts:

  • A line containing HTML version information
  • A declarative header section (delimited by the <head> element)
  • A body, which contains the document's actual content, implemented by the <body> element.

W3C

The declarative header section

In here you load up your custom cascading style sheets and other links you wish to add, for example: jQuery, bootstrap libraries are a few. You include the script libraries using the <script> tag, stylesheets, however, use the <style> tags or, alternatively, <link> tags. You are absolutely NOT allowed to put any other content in here.

The document's body

In here all of the content the user is subjected to is visible. You are free to use (and very much so, advised to use) various structure tags that HTML possesses. More advanced web developers use JavaScript in order to display or get some content from the page. The appearance of body's content can be influenced by the information given in the <head>.

TAG EXAMPLES
<a> Encloses text in a link.
<strong> Makes text bold.
<blockquote> Wraps the text into a quote.
<div> Extremely useful. Adds your content into a box that you can easily customise.
<form> Form component to access user input information.
<hr> Adds a long line to separate content.
<img> Adds an image to the page.
<p> Extremely useful. Just like div adds text into a box, although this is only for text paragraphs.
<ul> Adds a bullet-list.

Tags

  • Most commonly wrapped around in pairs, like so <p> ... </p>.
  • Some are just self-closing, because they have no contnet inside: <br />, <hr />.
  • Some tags can be nested within each other, like so: <p>... <em>...</em> </p>.

Common attributes

  • Class (used for many identical elements, to which you apply the same style)
  • ID (an unique element, one of a kind)
  • Style (provides inline CSS style rules)
  • Title (describes an element and its content, creates tooltips)

LESSON 2

Body tags

  • Block-level
    • Define blocks of content
    • Browser adds a line above and below
    • Default width and heigth
  • Inline
    • Does not add new lines
    • Must be inside a block element
    • Adopts the semantics of tjhe block-element it is in
  • Replaced
    • Has to be nested as an inline tag
    • Used for specific elements, such as images or form controls
    • Browser calculates dimensions and replaces by usually embedded objects

Block-level tags

  • Used for the basic structure of the web page
  • Text structure
    • Headings
    • Paragraphs
    • Horizontal rules
  • Tables
    • To present data accurately
    • Can be used for (not advised) to layout content (rather use Bootstrap or such)
