LESSON 1

Net and the Web

Net is a network of networks. It is well explained with an allegory of sending packets of information. As a packet is like a postcard with an address on it, given to another computer receiver, which is also connected as a part of the Net, each computer is capable of figuring out which cable should be used in order for the packet to reach its destination. That's the Internet's primary usage. Notable is that it operates at extraordinary speeds, delivering these packets well under a second.

The Web, however, is an abstract notion of an inanimate space, which holds information. If Net consists of painstakingly wired computers, Web holds documents, sounds, videos - entertainment! The cable connections are replaced with hypertext links. This inanimate space only exists because of their Net parents who communicate between numerous cables.

Blum on the Internet

We should all learn to be information smart: to understand when a Web site, or a piece of software, or an Internet Service provider plan, is giving us biased information. We should learn to distinguish quality information and quality links. As technology evolves, and machine-understandable information on the Web becomes available, we should be aware of the sudden changes which large-scale machine processing might have on our businesses.

The Memex machine, World Wide Web prototype

Some of the first trials at creating somewhat of an Internet prototype was a Memex machine created by Vannevar Bush in 1945. It was described as a device in which various books, records, communications and so on would be stored, "mechanized so that it may be consulted with exceeding speed and flexibility". To illustrate this we can imagine lists of static microfilm pages. However, this ingenious innovation lacked an actual hypertext system that would link pages together, therefore one could look at multiple images for a long while eventually realising he/she wasn't looking at the right sheet all along.

INTERNET

Networking infrastructure that allows hardware devices to physically connect to one another

  • Servers, Desktop PCs, Mobile devices etc.
  • Based on common standards – notably the TCP/IP protocol suite.

Protocols specify how the transactions are carried out

  • How communication is initiated and maintained
  • Type, size and structure of the data sent/received
  • Enable the services running over the internet (the Web is just one!)

Software clients (and servers) allow users and devices to access and use internet services

  • Using centralised servers or distributed peer-to-peer systems (where the wall between client and server is blurred)

TCP/IP: The heart of the Internet

TCP (Transmission Control Protocol)

Direct connection between client and a server, takes a message (give me a homepage), breaks it into smaller chunks, which can be sent to the IP protocol. At the receiving end TCP is responsible for taking care of these chunks and reassemble them so the application running them knows what to do with them.

IP (Internet Protocol)

Responsibility of taking packets and getting them to the destination, does not concern itself with how the packets are constructed, but only how to get them where needed. Also gives machines on the internet unique addresses. IP addresses point to servers, individual machines.

Packets

  • Header (addressing/sequencing information and metadata)
  • Body (actual data payload)

IP routing is connection-less. If you're sending a large request, transmitting would take 30 seconds. IP makes it unnecessary to have a stable connection for the whole 30 seconds. Liberates us a bit as well to take multiple routes to reach the end point. Multiple switches, cables routes to get to the same place.

ADVANTAGES

If one route is slow, then we can go around and take another. One wire can accommodate not only my messages but also other peoples'.

DISADVANTAGES

These things get corrupted, things arrive out of sequence, duplicate themselves, get lost completely.

IP is best-effort-delivery(?)

  • False hardware or software, delays can mess up with the performance of IP routing.
  • IP can't guarantee the wholeness or overall transport of the information.
  • TCP sits on the server and takes the responsibility to pull the packets off of IP and make sense of them, the header carries their sequence. It can reach back out and request to send that lost bit again. One out of the whole package. It can also refuse to accept or wait anymore for the bits, drop connection.

Domain names (DNS)

Map from one of the IP addresses to a domain name. When one types a URL into a browser, browser will look internally and tell if it had been to the website before. If it finds nothing, it will reach to a domain name server (there are thousands, like a phonebook) and issues a query. That's how one reaches from a domain name to an IP. There are thousands of servers on the internet that communicate between themselves if some domains move between different IP addresses.
